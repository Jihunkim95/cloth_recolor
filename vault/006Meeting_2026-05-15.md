# 미팅 2026-05-15

## 1. 9페이지 원래 아이디어 (soft-label calibration)

### 문제 정의

Gaussian 의 cloth label 이 현재 **hard fix** (0/1):
- `is_cloth: True or False`
- *"정답을 인간이 줄 수 없음"* — GT 없음
- 그래서 **soft label** 정보 추가 필요

### Soft-label 학습 공식 (hard label calibration)

학습 시:
- `C = 1` → `C = 1 − β · feature(x_point)`  ← 옷 쪽 점을 약간 낮춤
- `C = 0` → `C = 0 + α · feature(x_point)`  ← 옷 아닌 쪽 점을 약간 높임

`α, β` 는 feature distance 기반.

### Soft-label 생성을 위해 무엇이 필요한가?

**특정 학습 iteration 마다**, 다음을 모음:

1. **Easy sample 식별** — 예측이 0 또는 1 에 매우 가까운 *확실한* 점들
   - Class 0 (옷 아님) easy: top-1000 개
   - Class 1 (옷) easy: top-1000 개
   - 메모리에 저장

2. **각 학습 point 의 calibration**:
   - 그 point 의 feature
   - Easy sample set 에서 *가장 가까운 점* 과의 feature distance 계산
   - **distance 가 멀수록** soft-label `α` 를 크게 곱해서 *애매하게* update
   - → calibration

### Feature distance 의 두 가지 정의

| 방식 | feature 정의 | distance 계산 |
|---|---|---|
| **(a) Gaussian param distance** | gaussian 의 parameters (xyz, scale, rot, opacity, SH) | parameter space 의 거리 |
| **(b) Image feature distance** | renderer 가 뱉는 **2D 이미지 patch 를 DINOv3 vision extractor** 로 추출 | SAM3 + DINOv3 feature space 의 거리 |

방식 (b): "옷" 가까운 patch sample 모으고, "옷 아닌" patch sample 모음 → SAM3, DINOv3 두 모델 결합.

### Method 명명

**"memory based negative mining 4D gaussian splatting training for instance label"**

### Application

soft label 잘 찾아내면:
- 색깔 editing
- watermark insertion
- 기타 instance-aware editing applications

## 2. 피드백 — *9페이지 문제와 달라졌음*

### 0) Baseline 비교 정의

| 방식 | 파이프라인 |
|---|---|
| **Baseline** | jumping jack 의 4DGS 결과 → SAM3 "hoodie" mask 씌우고 색 변경 |
| **우리 방법** | 4DGS 에 cloth label 두고 *추가 학습* 해서 더 좋아진 것 |

### 0.1) Cloth label GT 가 없음

→ "얼마나 옷을 더 잘 찾는지 **수치적 체크 불가능**" (no quantitative metric)

### 0.2) SAM3 속도 분석

질문: SAM3 가 얼마나 걸리느냐? 미리 SAM3 돌려두고 4DGS 렌더링 시 옷을 바로 찾아내 두는 것이 **속도** 에서 어떤 이득?

**파이프라인 비교**:

```
baseline (infer):
   4DGS → SAM3 cloth detection → color change → finalized RGB

김지훈 (train):
   SAM3 with cloth     ← 학습 시 한 번
   ──────────────
   (inter):
   4DGS + cloth → color change → finalized RGB   ← 추론 시 SAM3 없음
```

→ 추론 시 SAM3 제거하는 *효율성* 이 contribution.

### 0.3) 2D pixel → 3D GS 자연스러운 mapping

SAM3 label 은 2D pixel 기준. 단순 매핑은 point 마다 0/1 hard:

**개선**: `soft_target_i = (True 갯수) / (SAM3_mask grid)`

현재 grid=16. *2D pixel object mask → 3D GS 학습 시 자연스러운 매핑* 이 핵심 질문.

#### 평가 방법

1. **Edge metric (수학)**: 옷과 다른 영역의 분리 라인에서 *RGB pixel edge* 와 *prediction value edge* 비교 → metric
2. **Human review (휴먼)**: 사람한테 보여주고 시각적으로 좋은지 0/1 투표
3. **Qualitative (정성적)**: 확대해서 보여주기

### 0.3+α) Label mapping baseline 실험

**baseline 단순 매핑 비교**:
- 점들에 0/1 매핑 → **0 초과 값 모두 1**
- 점들에 0/1 매핑 → **0.5 초과 값만 1**

**확장**:
- cloth 외 다른 object: 신발, 모자, ... → `object type N × NeRF Modeling M`
  - 최소: N ≥ 3 (cloth, shoes, hat ...), M ≥ 2 (scene 종류)
- `soft_target_i = True 갯수 / SAM3_mask grid` 는 **frame 1 장면** 용
  - → 4D 축으로 늘릴 수 있으면? (time frame grid 추가, optical flow 로 object 의 frame 별 위치 추정)
  - → **4D 를 위한 label mapping** (학습도 해야 함)

### 9페이지 원래 plan

positive/negative mining 을 통한 *"training softness"* — 위 모든 논의의 출발점.
