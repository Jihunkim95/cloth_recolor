## 2026-05-12 — [[results/exp010_per_gaussian_projection|exp010]] backprop 흐름 + 3D vs 2D 편집

### Q1. backprop 에 우리 알고리즘이 들어가는가?

**부분적 — 들어가지만 형태가 다름**.

[[results/exp010_per_gaussian_projection|exp010]] 의 4 stage 별 backprop 사용:

| Stage | backprop? | 우리 알고리즘 들어가는 곳 |
|---|---|---|
| 1. Vanilla 4DGS 학습 | ✓ 표준 backprop | 우리 알고리즘 X — RGB photometric L1+SSIM |
| 2. per-Gaussian soft_target 계산 | ✗ **gradient 없음** | ✓ 여기가 우리 핵심 알고리즘 (forward-only) |
| 3. cloth_logit [[notes/bce_loss\|BCE]] 학습 | ✓ 표준 backprop (1D scalar per Gaussian) | gradient 식 = ∂BCE/∂ℓᵢ — 표준 |
| 4. HSV recolor | ✗ inference only | 우리 알고리즘 X (HSV 변환은 표준) |

#### Stage 2 가 왜 special

우리 contribution 은 **"새 gradient 식" 이 아니라 "새 target 만드는 방법"**:

```python
# Stage 2 (gradient 없음, forward lookup only)
for each Gaussian g_i:
    soft_target_i = 평균 over frames(SAM3_mask[project(deform(g.xyz, t), cam_t)])
# ↓ 이 soft_target 이 Stage 3 의 supervision 신호로 들어감
# ↓
# Stage 3 (표준 BCE backprop, soft_target 은 constant)
for step in 2000:
    pred = sigmoid(cloth_logit)
    L = BCE(pred, soft_target)   # ← soft_target 은 detach 된 상수
    L.backward()                  # ← cloth_logit 만 gradient 받음
```

→ 표준 BCE 학습에서 *target 자체를 우리 방식으로 미리 만들어 둔 것*. backprop 알고리즘은 손대지 않음.

비유:
- 일반 학습: "정답 라벨이 (사람이 표기한) hard 0/1"
- 우리 방식: "정답 라벨을 (projection 으로 계산한) soft [0, 1]"
- backprop 식은 동일 (BCE). **데이터 (target) 만 우리가 생성**.

#### 왜 Stage 2 가 backprop 안 들어가나 (의도된 설계)

deformation MLP forward + camera projection + mask lookup 은 *미분 가능* 한 연산이지만 의도적으로 끊음:

- `pc._xyz.detach()`, deformation MLP 의 모든 param `.requires_grad_(False)` 로 freeze
- soft_target 계산 결과를 `.detach()` 해서 constant 화

→ Stage 3 에서 backprop 이 soft_target 으로 흘러가지 않음. cloth_logit 만 학습 변수.

**이유**: base 4DGS 의 학습된 기하학 (xyz, scale, deformation MLP) 을 *전혀 건드리지 않고* cloth label 만 추가하기 위함. RGB 품질을 깨뜨릴 위험 제거.

### Q2. 3D Gaussian 의 H 가 변하는가, 렌더링된 2D 이미지가 변하는가?

**3D Gaussian 의 SH_DC (=베이스 색) 가 직접 변함**. 2D 이미지는 그 결과로 새로 렌더되어 변함.

#### 정확한 흐름

```
[학습된 4DGS]
가우시안 i 마다 attribute:
  xyz, scale, rotation, opacity, SH_DC (3 float), SH_rest (45 float), cloth_logit

[Recolor (Stage 4)]
for i where sigmoid(cloth_logit_i) > τ:
    rgb_i_old = SH_DC_to_RGB(pc._features_dc[i])     ← 현재 베이스 색
    hsv_i = rgb_to_hsv(rgb_i_old)
    hsv_i[0] = 220/360                                ← H 만 교체 (S, V 유지)
    rgb_i_new = hsv_to_rgb(hsv_i)
    pc._features_dc[i] = RGB_to_SH_DC(rgb_i_new)     ← ★ 3D Gaussian 의 SH_DC 가 변함
    # SH_rest 는 손대지 않음 (specular/광택 패턴 유지)

[Render]
render(pc, viewpoint_t) → 2D 이미지 (이전과 다른 색)
```

코드 위치 — `2_process/recolor.py`:
```python
pc._features_dc[cloth_mask, 0] = new_sh_dc_t[cloth_mask]
# ← 이 한 줄이 핵심. pc._features_dc 는 3D 가우시안의 SH_DC tensor (N, 1, 3).
#   cloth_mask 가 True 인 행 (cloth 가우시안) 의 SH_DC 만 새 RGB 로 덮어씀.
```

이후 매 frame 마다 `render(pc, viewpoint)` 호출하면 변경된 SH_DC 가 alpha-composit 으로 자동 반영되어 2D 이미지 생성.

#### 비교: 2D 이미지를 직접 바꾸면?

| 옵션 | 효과 | 한계 |
|---|---|---|
| **2D image 만 변경** (예: pixel-level color edit) | 한 frame 만 변경 | 다른 시점/시간 보려면 매번 재편집 필요 — 시간/시점 일관성 X |
| **3D Gaussian SH_DC 변경** (우리 방식) | 한 번 변경 후 어느 시점·시간 렌더해도 자동 적용 | 시간 일관성·시점 일관성 자유 |

#### 비유

- **3D Gaussian 색 변경** = 옷 자체를 *파란 페인트* 로 다시 칠함 → 어느 각도/시간 에 봐도 파랑
- **2D 이미지 색 변경** = 한 장의 *사진* 에만 파란색 칠함 → 다른 각도/시간 사진은 그대로

우리 결과 (`3_output/<scene>/recolor_exp010_per_gaussian/recolored.ply`) 는 *첫 번째* — 변경된 3D scene 자체. viewer 에서 frame slider 돌리거나 카메라 회전해도 옷은 파란색 유지.

### Q3. plane 에 투영했으면 plane → gaussian backprop 해야 하지 않나?

**좋은 지적 — 의도적으로 그렇게 안 했음**.

#### 두 설계 옵션

**Option A** — Plane→Gaussian backprop (= [[results/exp001_tier2_dnerf_baseline|exp001]]~[[results/exp009|exp009]] 방식)

```
[forward]
cloth_logit → gsplat alpha-composit → 2D cloth_logit_image
                                          ↓
                          SAM3 mask 와 per-pixel BCE
[backward]
픽셀별 gradient
   ↓ (rasterizer 의 미분)
픽셀 → 그 픽셀에 splat 한 모든 가우시안의 cloth_logit 으로 흘러감
```

이게 통상적인 "plane→gaussian backprop". **문제**: 한 가우시안 ↔ 여러 픽셀 → gradient 가 평균 → **alpha-composit averaging 문제**. 옷 경계 가우시안이 옷 픽셀과 인접 피부 픽셀 양쪽에 splat → gradient 평균 0.4~0.6 → over-cover.

**Option B** — exp010 의 우회 (plane→gaussian backprop *생략*)

```
[Stage 2 — forward only, gradient 없음]
for each Gaussian i:
    g_t = deformation(xyz_i, t)               ← detach
    px, py = project(g_t, cam_t)              ← detach
    soft_target_i = SAM3_mask[t, py, px]      ← lookup, gradient 없음
    [모든 t 평균]

[Stage 3 — backprop 표준 BCE]
pred = sigmoid(cloth_logit)                    ← cloth_logit 만 학습 변수
L = BCE(pred, soft_target)                     ← soft_target 은 상수
L.backward()                                    ← cloth_logit 만 gradient 받음
```

projection 결과 (soft_target) 가 gradient 흐름과 **단절**됨. backprop 은 cloth_logit ↔ BCE 사이에서만.

#### 왜 그렇게 했는가 — 두 가지 이유

**1. cloth_logit 은 projection 에 등장하지 않음**

Stage 2 projection 에 쓰이는 변수: `xyz, deformation MLP, camera, mask`. 학습 변수 `cloth_logit` 은 여기 등장 안 함.
→ projection 통해 backprop 해도 *cloth_logit 에 학습 신호 흐를 길이 없음*. 어차피 무의미.

```
∂soft_target / ∂cloth_logit = 0
→ projection chain 통해 backprop 흐를 필요 없음
```

**2. base 4DGS 의 기하학을 보호**

만약 projection 통해 backprop 흐르게 하면 xyz / deformation MLP 가 update → 학습된 RGB 품질·기하 깨질 위험.

→ "학습 1: 기하 (RGB), 학습 2: semantic (cloth_logit), 분리" 의 의도. transfer learning 에서 pre-trained backbone freeze + head 만 학습하는 철학과 같음.

#### 핵심

backprop 은 **"학습 변수 → loss 사이"** 에서만 의미.
exp010 의 학습 변수 = `cloth_logit` → loss = `BCE(sigmoid(cloth_logit), soft_target)`.
soft_target 안에 cloth_logit 없음 → projection chain backprop 불필요.

| | gradient 흐름 | 결과 |
|---|---|---|
| exp001~009 (per-pixel BCE) | plane → gaussian backprop, alpha-composit averaging 포함 | over/under cover |
| **exp010 (per-Gaussian target)** | **projection 은 forward-only, BCE 만 backprop** | **깔끔한 cloth 분리** |

### Q4. 투영했을 때 3D Gaussian 이 SAM3 mask 영역에 있는지 어떻게 알아?

**Camera projection matrix 로 2D 픽셀 좌표 계산 → 그 픽셀의 binary mask 값 lookup**. 행렬 곱 + 배열 인덱싱.

#### 단계별 흐름

각 가우시안 $i$ 의 canonical `xyz` + 각 frame $t$ 의 `full_proj_transform` 으로:

```python
for each frame t:
    # (a) 시간 t 의 가우시안 위치 (deformation MLP 적용)
    g_t = deformation_MLP(canonical_xyz, t)        # (N, 3)

    # (b) Homogeneous coord → camera clip space 로 변환
    ones = torch.ones((N, 1))
    homo = torch.cat([g_t, ones], dim=1)           # (N, 4) = [x, y, z, 1]
    clip = homo @ camera_t.full_proj_transform     # (N, 4) = [x', y', z', w']

    # (c) Perspective divide → NDC ([-1, 1] cube)
    w = clip[:, 3:].clamp_min(1e-6)
    ndc = clip[:, :3] / w                           # (N, 3)

    # (d) In-view check (frustum 안인가)
    in_front = clip[:, 3] > 0                      # 카메라 앞
    in_view = in_front & (ndc[:, 0].abs() < 1) & (ndc[:, 1].abs() < 1)

    # (e) NDC → pixel 좌표
    px = ((ndc[:, 0] + 1) * 0.5 * W).long()       # (N,) ∈ [0, W)
    py = ((ndc[:, 1] + 1) * 0.5 * H).long()       # (N,) ∈ [0, H)

    # (f) SAM3 mask 값 lookup (2D 배열 인덱싱)
    mask_value_per_gaussian = SAM3_mask_t[py, px]  # (N,) ∈ {0, 1}

    # (g) In-view 인 가우시안만 누적
    accum += where(in_view, mask_value_per_gaussian, 0)
    count += in_view.float()
```

최종:
```
soft_target_i = accum_i / count_i  ∈ [0, 1]
```

#### ASCII 도식

```
[3D 공간]                              [2D 이미지 + SAM3 mask]

  •  Gaussian g_i 의 중심                 ┌──────────────┐
      \   (xyz_t = deform(xyz, t))        │░░░░░░░░░░░░│  ← mask = 0 (배경)
       \                                  │░░░╬╬╬╬░░░░│
        \                                 │░░╬╬╬╬╬╬░░░│  ← mask = 1 (cloth)
         \                                │░░╬╬╬•╬╬░░│      px=87, py=120
          \  full_proj_transform          │░░╬╬╬╬╬╬░░░│       ↑
           \                              │░░░░░░░░░░░░│   projection 결과
            ─────────────────────→        └──────────────┘
                                          mask_value = SAM3_mask[120, 87] = 1
                                          → g_i 의 이 frame 에서 cloth label = 1
```

#### "Gaussian 이 mask 영역에 있나" 의 정확한 의미

우리가 체크하는 것:
- **가우시안의 중심점 (mean xyz)** 이 2D 픽셀 좌표 (px, py) 로 매핑됨
- 그 픽셀의 SAM3 mask 값 (binary 0/1) 을 lookup

→ **"Gaussian 중심의 1픽셀"** 이 mask=1 영역에 있나만 체크. Gaussian 의 크기 (scale) 나 footprint (splat 반경) 은 고려 X.

이건 의도된 단순화 — *대표점 (mean)* 으로 충분히 추정 가능. 200 frame 평균 내면 noise 사라짐.

#### Frame 별 결과 누적 → 가우시안 라벨

- $g_i$ 가 hoodie 위 → 매 frame mask=1 → 평균 ~1.0
- $g_i$ 가 face 위 → 매 frame mask=0 → 평균 ~0.0
- $g_i$ 가 hoodie 경계 → 평균 ~0.3-0.5 (ambig)

이게 곧 `soft_target_i`.

#### 핵심 한 줄

**"3D Gaussian 의 중심을 카메라 행렬로 2D 픽셀에 매핑 → 그 픽셀의 SAM3 mask 값 (0 or 1) 을 읽어옴 → 200 frame 평균 = 그 가우시안의 cloth 일 확률"**.

### Q5. soft_target 을 BCE 의 supervision 으로 공급했으면, soft_target 을 BCE backprop 한 거 아닌가?

**"backprop 한다" 의 주어가 누군지 분리해야 함**. soft_target 은 **공급된** 것이지 **업데이트된** 것이 아님.

#### BCE backprop 의 정확한 의미

BCE loss 식:

$$
L = -\big[\, t \cdot \log \sigma(\ell) + (1 - t) \log (1 - \sigma(\ell)) \,\big]
$$

- $t$ = `soft_target` (projection 으로 계산한 상수)
- $\ell$ = `cloth_logit` (학습 변수)

Backprop 은 **학습 변수 $\ell$ 에 대해 gradient 를 계산**:

$$
\frac{\partial L}{\partial \ell} = \sigma(\ell) - t
$$

→ gradient 는 **$\ell$ (cloth_logit) 으로만 흐름**. $t$ (soft_target) 에는 안 흐름 ($t$ 가 상수니까).

#### 두 가지 다른 의미

| 의미 | 우리 알고리즘 | 비유 |
|---|---|---|
| **A. "soft_target 을 BCE 의 supervision 으로 *사용*"** | ✓ 맞음 | "정답지를 학생에게 보여줌" |
| **B. "soft_target 자체가 backprop 으로 *업데이트*"** | ✗ 아님 | "정답지를 학생이 고쳐 쓴다" |

질문은 A 와 B 를 합쳐 표현 → 분리해야 명확.

- soft_target 을 *공급* (forward 으로 만들고 BCE loss 안에 넣음) ✓
- 그 결과 cloth_logit 이 *업데이트* (backprop) ✓
- soft_target 자체는 절대 변경 안 됨

#### 코드로 보면 명확

```python
soft_target = compute_via_projection(...).detach()    # ← .detach() 가 핵심
                                                       #   gradient 끊김

optimizer = Adam([cloth_logit], lr=0.05)              # ← 학습 대상 = cloth_logit 뿐

for step in 2000:
    pred = sigmoid(cloth_logit)
    L = BCE(pred, soft_target)                         # ← target 으로 사용
    L.backward()                                        # ← grad 계산
                                                       #   soft_target.grad = None
                                                       #   cloth_logit.grad ≠ None
    optimizer.step()                                    # ← cloth_logit 만 업데이트
```

`.detach()` 가 명시적으로 "이건 학습 대상 아니야, gradient 계산 X" 를 선언.

#### 비유 — 시험과 채점

```
학생         = cloth_logit (학습 변수)
답안 작성     = sigmoid(cloth_logit) = prediction
정답지       = soft_target (projection 으로 미리 만든 supervision)
채점         = BCE loss
공부 (학습)   = backprop + Adam → 학생이 답안 작성 능력 개선
```

- "정답지를 채점에 사용했다" (✓ A)
- "정답지를 backprop 으로 고쳤다" (✗ B, 아님)

정답지 (soft_target) 는 시험 시작 *전* 에 만들어진 고정 자료. 채점은 학생 답안과 정답지를 *비교* 만 함. **학생만 학습**.

#### 영어 표현 정정

- ✗ "backprop the soft_target through BCE" — 의미 없음
- ✓ "use soft_target as target in BCE to train cloth_logit by backprop"

#### 핵심 한 줄

> **backprop 은 항상 *학습 변수* 에 대해서만 일어남**. soft_target 은 supervision 신호 (= 정답지), 학습 변수 cloth_logit 이 backprop 으로 update. 두 역할 분리.

### Q6. cloth_logit 은 Gaussian 의 추가 파라미터로 들어가는 거지?

**맞음. 각 가우시안마다 1 개 scalar 가 추가됨**.

#### 가우시안 1 개의 attribute 목록

이전 (4DGS):
```
가우시안 i:
  xyz_i         (3) ← 위치
  scale_i       (3) ← 크기
  rotation_i    (4) ← 회전
  opacity_i     (1) ← 투명도
  SH_DC_i       (3) ← 베이스 색 RGB
  SH_rest_i    (45) ← view-dependent 색 (광택)
```

우리가 추가:
```
  cloth_logit_i (1) ← ★ "옷일 확률 logit"
```

총 60 → 61 float per 가우시안. 1 float 만 추가.

#### 다트 비유 확장 — N 명의 게임

이전 답변에서 "아이 1명" 이라 했지만 실제로 **가우시안마다 아이 1명** 동시 학습:

```
가우시안 N = 25,000개 (jumpingjacks 예시)
→ 아이 25,000명 동시 학습

가우시안 1:    [아이 1]   ──다트── ▶ [표적 1: 0.91 = 옷일 확률 매우 큼]
가우시안 2:    [아이 2]   ──다트── ▶ [표적 2: 0.03 = 옷 아님]
가우시안 3:    [아이 3]   ──다트── ▶ [표적 3: 0.45 = 애매]
...
가우시안 25k:  [아이 25k] ──다트── ▶ [표적 25k: 0.88]

각 아이의 자세  = cloth_logit_i
각 아이의 표적  = soft_target_i (projection 으로 미리 계산)
backprop       = 25,000 아이들 자세 동시 교정 (각자 자기 표적 맞게)
```

각 아이가 *독립적으로* 자기 표적 맞추도록 학습. BCE 가 가우시안마다 따로 계산되므로 서로 영향 X.

#### 영향 끼치는 3 곳

**(1) PLY 파일 안 (저장)**

학습 끝나면 `point_cloud.ply` 안에 컬럼 1 개 더 생김:
```
ply
  property float x        ← xyz
  property float y
  property float z
  property float f_dc_0   ← SH_DC
  property float f_dc_1
  property float f_dc_2
  property float f_rest_0
  ...
  property float opacity
  property float scale_0
  ...
  property float rot_0
  ...
  property float cloth_logit   ← ★ 새 컬럼
```

**(2) 학습 시점 (`train.py`)**

옛날 ([[results/exp001_tier2_dnerf_baseline|exp001]] ~ exp009): 픽셀 BCE 방식이라 cloth_logit + 모든 attribute 같이 학습됨 → 기하 흔들림.

[[results/exp010_per_gaussian_projection|exp010]] 에서 바꿈:
- `xyz, scale, rotation, opacity, SH, deformation MLP` 모두 freeze (학습 X)
- **cloth_logit 만 학습**

```python
# Stage 3
for param in [xyz, scale, rotation, opacity, SH_DC, SH_rest, deformation_MLP]:
    param.requires_grad = False    # ← 얼림

cloth_logit.requires_grad = True    # ← 이것만 학습
optimizer = Adam([cloth_logit])     # ← optimizer 에도 cloth_logit 만 등록
```

**(3) Recolor 시점 (`recolor.py`)**

학습된 cloth_logit 을 읽어 "옷 가우시안" 골라냄:
```python
cloth_mask = sigmoid(cloth_logit) > τ    # ★ cloth_logit 가 여기 쓰임
for i where cloth_mask[i]:
    SH_DC_i = HSV_swap(SH_DC_i)          # 옷 가우시안의 색만 변경
```

#### 핵심 한 줄

> `cloth_logit` 은 **가우시안마다 1 개씩 추가한 학습 가능 scalar**. 다른 attribute 는 그대로 (xyz, scale, ...). Stage 3 에서 이 한 채널만 backprop 으로 학습. 학습 끝나면 `sigmoid(cloth_logit) > τ` 인 가우시안의 색만 변경.
